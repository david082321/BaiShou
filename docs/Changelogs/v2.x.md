###### v2.3.0 「数据库架构优化与一致性信号」 _(2026-03-07)_

> **里程碑**：重构了数据库的物理拓扑，并建立了全局数据刷新总线，极大提升了多 Vault 场景下的稳定性。

- **数据库物理迁移**：将 `baishou.db` 移入各 Vault 根目录下的 `.baishou/` 隐藏文件夹。实现了数据库与原始数据的物理解耦，彻底解决了多库并存时的索引冲突。
- **全局数据刷新信号 (dataRefreshProvider)**：建立基于 Riverpod 的全局版本计数器。导入数据、切换 Vault 或执行清除操作时，一键通知所有 UI 组件（主页、统计、总结列表）同步重刷。
- **数据清除逻辑增强**：重写"一键清除数据"逻辑，采用文件夹递归粉碎模式，确保出厂设置重置的彻底性。
- **导入安全性加固**：针对 S3 或 ZIP 导入流程，引入了"等待扫描锁定槽"机制，防御多线程并发写入带来的并发异常。

###### v2.2.0 「文件夹热切换与稳定性修复」 _(2026-03-06)_

> **里程碑**：修复了更换存储文件夹后必须重启方能重新扫描的问题，实现了真正的"目录热切换"。

- **目录热切换**：修复了 `FileStateScheduler`、`SummarySyncService` 等核心单例服务的 Vault 监听逻辑——以前只判断库的"名称"是否改变，更换根目录时因为库名不变而被全部忽略。现在增加了"物理路径"的比对，路径一旦变化便立即重新绑定文件监控器并触发全量扫描。

###### v2.1.0 「总结物理化与数据安全加固」 _(2026-03-05 ~ 2026-03-06)_

> **里程碑**：实现了总结数据（AI 回忆）的完全物理化存储，彻底兑现"数据绝对主权"的承诺。并深度加固了底层同步引擎，消灭了并发竞态带来的数据威胁。

- **总结物理化**：AI 生成的周/月/季/年报全面拥抱文件系统，在对应的 Archives/ 目录落盘为携带时间属性的 Markdown 文件。实现与日记同等的数据迁移随身性。
- **无感迁移机制**：首创一次性自校验迁移逻辑体验，针对 v2.0.0 遗留的 SQLite 总结数据，在系统层级执行后台安全抽取，稳健完成新老版本的格式跨越。
- **竞态条件彻底防御**：修复了严重的并发截断 Bug。引入严格的系统级互斥锁 `_isMigrating`、`suppressPath` 及全量扫描入口强制状态校验，彻底免疫初始化顺序及操作系统的"文件回声"引发的数据库误删。
- **Android 权限预置**：优化了移动端挂载存储的体验流，将繁琐的主页"被动权限检查"改为 Onboarding 流程与设置内的"动作发生时主动系统授权"，实现"选完即用"。
- **导入优化**：完善了从 ZIP 备份导入时的元数据校验逻辑，修复了日记导入时丢失时间戳标题的视觉缺陷。

###### v2.0.0 「物理引擎与绝对数据主权」 _(2026-02-26 ~ 2026-03-05)_

> **里程碑**：全面进化为「本地 Markdown 文件 + SQLite 影子索引 + UI与内存绑定」三轨存储架构，主轴性能极大提升。确立了以 `DataArchiveManager` 为核心的数据中枢，彻底重构了局域网传输、云同步与多端迁移流程。

- **存储三轨化重构**：放弃纯数据库依赖，日记永久化为标准 `.md` 文本（携带 YAML 头映射天气、心情等元属性），仅将 SQLite 作为只读的一级高速缓存 (`ShadowIndexSyncService`)。
- **向后兼容与无缝迁移**：大幅重构 `ImportService`与 `DataSyncService`，原生兼容旧版 v1.1.0 的局域网与 ZIP 包迁移，在内存中动态提取旧源再重新以新格式落盘 Markdown，且完美保留时间轴属性。
- **数据中枢加固与同日合并**：建立 `DataArchiveManager` 统一管理快照与导入导出，修复了异构生命周期中的崩溃风险。针对 v1.1.0 的多记录历史，**新增了同日多篇日记自动合并特性**，通过五级标题（H5）时间戳对旧数据进行无损缝合，确保跨版本迁移的正文完整性。
- **数据库异构容错与自愈**：建立了完善的 SQLite 兼容性防线。针对跨设备迁移（ZIP导入）可能引发的 FTS5 模块缺失及数据库死锁，实现了物理粉碎损坏索引并原地触发无状态自愈重建的容错策略。
- **存储监听与拓扑对齐**：重构 `FileStateScheduler` 监听逻辑，将监控锚点提升至 Vault 根目录级别。实现了对 `Journals` 目录整体删除、恢复、移动等极端拓扑变动的感知。
- **Android "零接触"极简签名**：抛弃物理 `.jks` 证书文件，搭建基于 `key.properties` `storeBase64` 的内存动态反编译打包签名流水线，CI/CD 管线同步对齐。
- **视觉盛宴与瀑布流主页**：主页全面升级并重构为 Masonry 瀑布流展示，全局引入一次性索引加载 (`VaultIndex`)，视觉美感产生质变。组件层面修复了移动端模型溢出的布局缺陷。
- **国际化与导航扁平化**：全模块接入 `slang` 本地化，支持四国语言。将"局域网传输"提升为一级导航，大幅缩短操作路径。同时移除"回忆"模块中冗余的"原始数据"按日期导出 Tab。
- **Latte 与开源誓约**：开源协议从 GPL 激进迁移至 **AGPL-3.0**，确立吸血鬼少女 Latte（白发金瞳）为白守看板娘，全套定制 UI、Icon 及 GitHub Banner 已全面铺设。
