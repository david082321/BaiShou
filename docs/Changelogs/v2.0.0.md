## [2.0.0] - 2026-03-05

> 「文件系统更新」

这是白守迄今最重大的一次架构跨越。我们抛弃了纯数据库驱动的老路，彻底拥抱了"本地 Markdown 文件 + SQLite 影子索引"的全新三轨制存储体系。从这个版本起，你的每一篇日记，都是一个真实存在于文件系统里的 `.md` 文件，永远不会被锁在任何软件的私有格式里。

### 🏗️ 架构重构

- **存储三轨化**：日记永久落盘为标准 Markdown 文件（含 YAML 头，存储情绪、天气、标签等元属性），SQLite 影子索引库仅作高速查询镜像，两者共同服务于内存中的 `VaultIndex` 实时视图。
- **数据中枢统一**：新建 `DataArchiveManager` 接管所有数据操作（导入、导出、快照、云同步、局域网传输）。重构后的入口更加清晰，且完全消除了异步生命周期中 Provider 被意外销毁的 `Disposed Ref` 崩溃隐患。
- **日记纵深追加**：同一天可追加多条独立记录，支持内容动态拼接。

### ⬆️ 向后兼容迁移

- 完整兼容从 v1.1.0 的局域网传输和 ZIP 包迁移。导入时在内存中动态解析旧格式数据，并以新的 Markdown 格式重新落盘，同时**完美保留所有历史时间轴属性**。

### 🎨 视觉与体验

- **瀑布流主页**：主页全面升级为瀑布流布局，引入月份外骨骼、星期徽章、精确时间戳等精致美工，视觉层次感大幅提升，并支持三栏/双栏/单栏响应式切换。
- **模型选择器修复**：修复了移动端长模型名称溢出截断的布局缺陷，优化了 `Flexible` 文本收缩策略。
- **导航扁平化**：将「局域网传输」从三级嵌套提升至设置首页的一级导航，大幅缩短操作路径。
- **界面精简**：移除「回忆」页面的「原始数据」按日期导出 Tab，相关功能已由 `DataArchiveManager` 统一接管。

### 🌐 国际化 (i18n)

- 全模块接入 `slang` 本地化引擎，支持**简体中文、English、日本語、繁體中文**无缝动态切换。
- 修复了日记卡片月份、总结标题等大量硬编码文本，实现国际化引擎的完整闭环。
- 感谢社区贡献者 **[@david082321](https://github.com/david082321)** 提交了繁体中文翻译的精准语境修正，让繁中版本的表达更加地道自然。

### 🔒 数据安全

- **自动快照保护**：在所有覆盖性操作（导入、局域网接收）前，系统会自动创建当前数据的完整快照，提供反悔的空间。
- **还原链加固**：还原流程增加了"清空物理脏文件"步骤，彻底解决覆盖导入后出现历史残留数据的问题。

### 🤖 工程与发布

- **Android 零接触签名**：使用基于 `storeBase64` 的签名，CI/CD 管线同步对齐。
- **Windows 多语言安装包**：Inno Setup 安装程序现在根据系统语言自动显示中英日本地化界面（安装、卸载、桌面快捷方式等），不再是单一简体中文界面。
- **看板娘 Latte**：确立白发金瞳吸血鬼少女 Latte 为白守官方看板娘，全套定制 Icon、GitHub Banner 已同步铺设。
- **开源协议升级**：从 GPL 迁移至 **AGPL-3.0**。
